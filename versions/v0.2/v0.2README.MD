🚀 New Features in V0.2
1. Reverse Scrambler Engine (scramble command)
A new engine block has been added to compute the inverse of any stored OLL or PLL algorithm.

The scrambler performs the following operations:

Removes formatting characters such as parentheses

Splits the algorithm into individual moves

Reverses the move order

Converts primes to non‑primes and non‑primes to primes

Preserves double moves (2)

Outputs a clean, ready‑to‑use setup scramble

This feature enables targeted practice by generating the exact scramble required to reach a specific case.

2. New Command Gate: scramble
A dedicated command has been added to the main loop:

Code
scramble
The user can select OLL or PLL, choose a case, and receive:

the original stored algorithm

the computed inverse scramble

This integrates seamlessly with the existing solve workflow.

3. Non‑Destructive String Cleaning
Before inversion, the engine removes parentheses and other structural characters to prevent parsing errors:

Code
clean_alg = alg.replace("(", "").replace(")", "")
This ensures compatibility with more complex notations.

4. Core Workflow Preservation
The original V0.1 functionality remains intact:

list displays all stored OLL and PLL cases

add dynamically routes new cases into the correct category

OLL lookup transitions into PLL lookup for solve simulation

JSON persistence remains stable and self‑healing

The new scrambler extends the system without disrupting existing behavior.

🧠 Development Notes: Concepts Applied in V0.2
Advanced String Manipulation
The scrambler relies on Python string operations such as:

.replace()

.split()

.join()

These methods enable the transformation of raw algorithm text into structured move lists.

List Operations
The inversion logic uses:

.reverse()

iterative reconstruction of move sequences

This allows the engine to flip algorithms accurately.

Algorithmic Logic Translation
Rubik’s Cube notation rules were encoded into programmatic conditionals:

prime → non‑prime

non‑prime → prime

double moves remain unchanged

This bridges physical cube logic with digital computation.

Engine‑Block Architecture
The Reverse Scrambler represents the project’s first modular “engine block,” establishing a pattern for future subsystems.
